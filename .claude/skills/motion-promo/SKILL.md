# motion-promo — пайплайн «данные проекта → премиум промо-ролик»

Повторяемый навык. Детали крафта — в Obsidian-вике «мои проекты/Моушен-видео (Remotion)/motion-engine (наш движок)/»: **04 — Категории + структура**, **05 — Крафт (тайминг/переходы/приёмы/эталоны)**, **03 — Тестирование**. Инструментарий движка — `CLAUDE.md`.

Работать **из папки `motion-engine`**. Двигаться по шагам, **показывать стиллы/кадры пользователю и согласовывать** — не гнать вслепую.

## 1. Разведка проекта (URL/HTML/скриншоты → данные)
- Playwright: открыть сайт, снять **реальные экраны** (desktop + мобильный вид под девайс).
- Палитра — `getComputedStyle`: `body` bg, accent (кнопки/градиенты — вытащить точные hex), шрифт.
- Тексты/плюсы — реальные с сайта (цифры, слоганы, фичи). **Ничего не выдумывать** — реальный визуал/копия ([[feedback_realism_over_abstraction]]).
- Ассеты: лого (SVG inline или файл), иконки — вытащить URL, скачать в `public/<brand>/` (публичные наборы — ок).
- ⚠️ **Закрыть браузер после** (`browser_close`) — на Windows headless-Chrome оставляет висящее пустое окно.

## 2. Выбор категории (роутинг в 3 вопроса → вика 04)
1. Есть UI продукта? Нет → Brand-Hero / Data-Stats. Есть → дальше.
2. Один флоу или весь продукт? Один → **Feature-Demo** (наш эталон). Много → Product-Tour. Нужен «почему важно» → Explainer.
3. Дистрибуция? Ship-day → Launch. <9с → Social-Teaser. Доверие → Testimonial/Comparison.
→ Категория даёт **блюпринт биты + типичную длину + приёмы** (таблица в вике 04).

## 3. Структура (framework + бюджеты → вика 04)
- Выбрать фреймворк: **Hook→Demo→Benefits→CTA** (дефолт для UI-продуктов) / show-don't-tell / PAS / BAB.
- Бюджеты (~25с): hook 10-12% · demo 60-70% · benefits 15-20% · CTA 12-16%.
- Первые 3с несущие; hook — продукт mid-action или bold-число, НЕ «мы рады представить» и НЕ статик-лого.

## 3b. Brief-артефакт (checkpoint — Analyzer emit)
После разведки+категории+структуры (шаги 1–3) **записать `src/<brand>/<brand>.brief.ts`** по
`src/v2/briefSchema.ts` (zod): `product{name,url}` · `category` · `framework` · `palette{accent,bgFamily,
glowAlpha}` (accent — точный hex продукта, **должен совпасть** с будущим `theme.<brand>.accent`) ·
`features[]{label,rank,isHero}` (ровно один `isHero`) · `heroBeat{feature,why}` (ОДИН эмоц. бит; `feature`
= один из `features[].label`) · `copy{headline,cta,lines}` (**verbatim** с продукта, не выдумывать) ·
`assets[]`. Это фиксирует разведку как данные (иначе живёт только в памяти сессии) и заземляет весь
пайплайн в реальную конкретику. Эталоны: `src/lumo/lumo.brief.ts`, `src/zarya/zarya.brief.ts`. Проверка:
`briefSchema.parse` + `brief.palette.accent === theme.<brand>.accent` (см. `src/v2/brief.test.ts`).

## 4. Бренд-пресет
- `src/theme/tokens.ts` → `theme.<brand>` (~30 строк): bg (dark), **один accent** (из палитры), textMuted, card/cardInner, gradient, glow. Шрифт (Inter/Manrope). **Один accent на нейтрали** — не мульти-hue (premium vs scammy, вика 05).

## 5. Сборка (`src/<brand>/`, паттерн «проект внутри движка»)
- **Pixel-faithful UI** как React-компоненты (виджеты/экраны), реальные иконки через `<Img staticFile()>` (**НЕ `<img>`** — footgun: не грузится на части кадров). Flex-дети → `minWidth:0` (footgun #11).
- **Sequencing обязательно**: `<TransitionSeries>` из сцен (Hook/Demo/Benefits/CTA) — наслаивание сцен невозможно by construction (footgun #12).
- **Переходы**: `slide({direction})` разных направлений + `springTiming` (НЕ `fade()` на крупном UI — `shouldFadeOutExitingScene` default false → наслаивание). `MagicMove` — 1 hero-match-cut на ролик. **Ban wipe** (VHS).
- **Реальный флоу**, не телепорт: tap → sheet выезжает → выбор → встаёт в поле. Показывать тапы (`TapDot`; курсор/focus-примитивы — в backlog вики 05).
- Grain + дышащий фон-глоу; но **не** постоянный дрейф везде (филлер).

## 5a. 3D-объект в кадре (если нужен)
Нужен реальный 3D-предмет (девайс/карта/продукт)? **Два источника, оба живые** — выбирать по объекту, не по привычке:
- **Blender** (`scripts/blender/*.py` → `public/models/*.glb`, грузится `useGltf`) — когда нужна
  **запечённая фото-текстура НА поверхности** (экран с реальным UI, лицевая сторона карты),
  сложная топология или уже есть `.blend`. Цена: бинарник в репо, async-загрузка (`delayRender`),
  цвета живут в Python отдельно от `theme/tokens.ts`, автотестами не покрыт.
- **Процедурная фабрика** (`src/models/<name>/`) — когда объект **hard-surface и описывается
  примитивами** (корпуса, девайсы, карточки, плашки). Синхронна (нет гонок загрузки), в ~90 раз
  легче GLB, берёт цвета из токенов через `options`, покрывается юнит-тестами. Цена: фото-текстуру
  на поверхность так просто не положить.

Контракт фабрики — `src/models/types.ts`: `createXModel(options): ModelHandles<Parts>`, где `parts` —
живые ссылки на анимируемые узлы/материалы. **Никогда `getObjectByName` в потребителе** (footgun #7).
Фабрика обязана быть **без DOM и без async** (никаких canvas-текстур/загрузки картинок) — иначе
теряется юнит-тестируемость и возвращаются гонки рендера.

Пайплайн «референс-фото → модель» описан в `.claude/skills/img2threejs/` (вендорен). Использовать
**выборочно**: `validate_sculpt_spec.py --strict-quality` как линтер спеки модели,
`make_comparison_sheet.py` для side-by-side ревью (годится и для GLB), `extract_pbr_evidence.py`
для палитр, `grimoire/` как рубрики качества. **Кодогенератор (`generate_threejs_factory.py`) не
использовать** — он транслирует спеку 1:1, а спеку всё равно пишешь ты; его выход тянет `any`,
canvas-текстуры и async-загрузку карт. Писать фабрику руками.

⚠️ Палитра из `extract_pbr_evidence.py` — это цвет **освещённых пикселей**, не albedo. Подставишь
как `baseColor` — свет применится дважды и модель выйдет выцветшей. Затемняй.

Проверка: `npm run check-fidelity <BenchComp> --reference <фото>` — силуэт против референса (мягко)
+ регрессия против принятого baseline. Не заменяет просмотр глазами.

## 6. Тайминг (frame-математика @30fps → вика 05) — самое слабое место, держать строго
- Принцип: **действие → ПАУЗА увидеть результат**. После появления UI — **пауза осмотра 30-45f** до первого тапа. После каждого выбора — **15-24f hold**. Demo-flow: первый показ медленно (150-210f на 4 шага), повтор −30%.
- **Длина бита ∝ важности**, варьировать ±30-40% (одинаковые биты = робот). Ускорять к климаксу, CTA — снова долго (landing-strip freeze 2-3с).
- Экран для чтения ≥30-45f; хедлайн 36-60f; лого/CTA 90-300f.

## 7. Проверка (пирамида → вика 03) — НЕ срезать углы
- `npm test` + `npm run test:layout` (браузер, overflow) + `npm run check-render <Comp>` (content/motion/loop).
- **Глазами прогнать ВСЕ стыки на mid-transition кадрах** (не только стиллы сцен) — класс «стилл проходит, в движении ломается» (footgun #10/#12). Выдёргивать кадры стыков из mp4 ffmpeg'ом.
- Sequencing durations-math тест (`*.test.tsx`).

## 8. Рендер + финал
- `npx remotion render <Comp> out/final/<name>.mp4 --gl=swangle` (пин для детерминизма).
- Сжать под веб если для портфолио/деплоя; бэкап старого.
- Показать пользователю, итерировать по фидбеку.

## Частые грабли (проверять всегда)
- fade на крупном UI = наслаивание → slide+spring. `<img>` → `<Img>`. flex без `minWidth:0` → overflow. Один accent, не радуга. Тайминг: не торопить (пауза осмотра), не монотонить (варьировать). Проверять стыки видео, не стиллы.
